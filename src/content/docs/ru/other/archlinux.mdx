## Подготовка установочного образа

:::note
Хотим подчеркнуть, что системы Linux можно установить практически куда угодно: на настольный компьютер, ноутбук, USB-накопитель или даже на телефон (если дистрибутив это поддерживает). Процесс установки примерно одинаковый, но есть различия, которые нужно понимать. Например, при установке Arch Linux как второй системы вам потребуется вручную настроить efibootmgr. Кроме того, есть различия в настройке двойной загрузки на машине с BIOS и UEFI. Поэтому руководства будут повторять определённые шаги, чтобы избежать путаницы с многочисленными инструкциями типа «Если у вас BIOS, делайте так…» и «Если у вас UEFI, делайте так…».
:::

Что вам понадобится для начала установки:

- USB-флешка
- ISO-файл системы
- Программа для записи ISO-образа
- Устройство, на которое будет установлен Arch

Установка будет выполняться как основная операционная система.

Давайте создадим установщик на USB-накопителе.

Сначала скачайте образ системы с официального сайта Arch Linux удобным для вас способом. Мы рекомендуем использовать Torrent-клиент.

Есть несколько программ, которые могут помочь в этом процессе:

- Rufus
- Balena Etcher
- UltraISO

В нашем случае мы будем использовать Rufus.
После загрузки образа откройте Rufus.

- Устройство: Выберите вашу **USB-флешку**.  
- Метод загрузки: Выберите **«Диск или ISO-образ»** и укажите путь к ISO-файлу Arch Linux.  
- Схема раздела: Выберите **MBR**.  
- Файловая система: Выберите **FAT32**.  
- Размер кластера: Оставьте **по умолчанию**.  

![rufus_arch](https://github.com/TeamDominant/wiki/blob/master/src/assets/archlinux/rufus_arch.png?raw=true)

:::danger
Все файлы на USB-накопителе будут удалены! 
:::

- Нажмите Старт. Во всплывающем окне выберите опцию с упоминанием **ISO**.

После завершения процесса подключите USB-флешку к машине, на которую хотите установить систему.  
Войдите в BIOS машины и отключите Secure Boot (вы можете включить его снова после завершения установки).  
Теперь мы можем загрузиться с USB. Когда машина загрузится с USB, вы увидите меню. Выберите первый пункт **Arch Linux install medium** и дождитесь загрузки системы.

![start_arch](https://github.com/TeamDominant/wiki/blob/master/src/assets/archlinux/Arch_install1.png?raw=true)

## Установка

### Настройка беспроводной сети

Если вы подключены через проводное соединение, просто проверьте, работает ли оно, с помощью следующей команды:

```bash
ping archlinux.org
```

Если вы получили ответ, всё в порядке. Если нет, значит необходимая служба не запустилась. Попробуйте перезагрузить машину или пересоздать установочный образ. Мы будем использовать утилиту iwctl.

Выполните команду:

```bash
iwctl
```

![iwctl](https://github.com/TeamDominant/wiki/blob/master/src/assets/archlinux/iwctl_settings.png?raw=true)

Затем введите следующие команды:

1. Список доступных беспроводных устройств:

```bash
device list
```

*Определите имя вашего беспроводного устройства (например, wlan0).*

2. Сканирование доступных сетей:

```bash
station wlan0 scan
```

3. Просмотр списка доступных сетей:

```bash
station wlan0 get-networks
```

4. Подключение к Wi-Fi сети:

```bash
station wlan0 connect {network}
```

*После этой команды вам нужно будет ввести пароль, если сеть защищена.*

5. Проверка состояния подключения:

```bash
station wlan0 show
```

Убедитесь, что сеть подключена.

```bash
ping archlinux.org
```

### Разметка диска

Существует две утилиты для разметки диска:

- **fdisk**
- **cfdisk**

Мы будем использовать `fdisk`, который проще в использовании и более дружелюбен для неопытных пользователей.

Сначала определим имя нашего диска.

:::danger
Рекомендуется отключить все ненужные диски, чтобы избежать путаницы и случайной потери данных из-за неопытности.
:::

Используйте следующую команду для определения имени вашего диска:

```bash
fdisk -l
```

Это выведет список подключённых дисков. 

Выполните следующую команду для настройки вашего диска:  

```bash
fdisk /dev/nvme0n1
```

Нам нужно создать три раздела:

| Раздел    |   Размер          | Тип               |
| :---      |     :---:         |              ---: |
| boot      | 1G                |  Efi-system       |
| swap      | 8G                |  linux swap       |
| root      | Оставшееся место  |  linux filesystem | 

Нажмите ++p++ для просмотра разделов на выбранном диске, если они есть.

Нажмите ++n++ для создания нового раздела. Номера разделов оставьте по умолчанию. Оставьте **Первый сектор** по умолчанию, нажав ++enter++, и выделите `+1G` для **Последнего сектора**. Мы только что создали раздел `boot`.

Выполните те же шаги для `swap`, выделив 4-8G (по желанию), и всё оставшееся место для `root`.

:::danger
SWAP — это раздел подкачки; обязательно создайте его, если у вашей машины менее 8 ГБ оперативной памяти.
:::

После создания всех разделов нажмите ++w++ для применения изменений.

### Archinstall

Отлично, мы закончили с разметкой диска. Теперь приступим к установке Arch со всеми компонентами.

**Введите** и **ждите**:

```bash
archinstall   
```

Нас встретит следующее меню.

![archinstall](https://github.com/TeamDominant/wiki/blob/master/src/assets/archlinux/archinstall.png?raw=true)

Давайте пошагово пройдём процесс установки Arch Linux с помощью установщика:

**Locale (Локализация)**

*Здесь вы можете добавить язык, который будет использоваться в системе.*

- Выберите Locale language и найдите нужный язык. Вы можете выбрать `ru_RU`, если нужно.

:::note
Для быстрого поиска можно нажать ? и ввести нужный язык, например en или ru.
:::

**Disk configuration (Конфигурация диска)**

- Выберите `Manual Partitioning`.

- Выберите ваш диск, в нашем случае `nvme0n1`.

- Выберите раздел 1G. Назначьте точку монтирования: `/boot`. Выберите mark/unmark для форматирования. Измените файловую систему на `fat32`.

- Выберите раздел со всем оставшимся местом на диске. Назначьте точку монтирования: `/`. Выберите mark/unmark для форматирования. Измените файловую систему на `ext4`.

- Нажмите `Confirm and exit`.

**Bootloader (Загрузчик)**

- Выберите `GRUB`

**Hostname (Имя хоста)**

*Имя используется как сетевое имя компьютера*.

- Введите **имя** для вашей системы (можете оставить archlinux).

**Root password (Пароль root)**

*Установите пароль для пользователя root*.

- Введите **пароль** и **подтвердите его**.

**User account (Учётная запись пользователя)**

*Добавьте новую учётную запись пользователя*.

- Введите **имя пользователя**.

- Введите **пароль** для пользователя.

*Вам будет предложено подтвердить добавление привилегий суперпользователя (sudo)*.

- Выберите `Yes` для подтверждения.

**Profile (Профиль)**

*Выберите графический интерфейс для вашей установки Arch*.

- Нажмите `Type`.

- Выберите `Desktop`.

- Выберите `KDE plasma`.

- Выберите `sddm`

Выберите рабочее окружение, которое вам подходит. Рекомендуется выбрать **KDE Plasma**.

Вы будете перенаправлены в следующее меню.

Вы можете выбрать подходящий графический драйвер в зависимости от конфигурации вашей системы. Для NVIDIA рекомендуется использовать проприетарный драйвер.

**Network configuration (Конфигурация сети)**

- Выберите `Use NetworkManager`

**Additional packages (Дополнительные пакеты)**

*Добавьте дополнительные программы, которые могут быть полезны*.

Пока добавим:

`micro nano firefox btop`

**Timezone (Часовой пояс)**

*Выберите ваш часовой пояс.*

:::note
Для быстрого поиска нажмите ? и введите нужный часовой пояс.
:::

**Audio (Аудио)**

Выберите `pipewire`.

**Installing Arch (Установка Arch)**

После предварительной настройки выберите `Install`.

Затем выберите `Yes` и дождитесь завершения установки.

После завершения установки нам будет предложено продолжить настройку в chroot. Выберите `Yes`.

### Конфигурация GRUB

*Теперь давайте настроим grub*. 

Как мы видим, arch отсутствует в менеджере загрузок, выполнив команду:

```bash
efibootmgr
```

Изменим конфигурацию:

```bash
grub-install --target=x86_64-efi --efi-directory=/boot --recheck
```

При успехе появится `Installation finished. No errors reported`.

Проверьте снова, должна появиться запись arch:

```bash
efibootmgr
```

Наконец: 
```bash
reboot
```

## Двойная загрузка

Если вам нужно добавить запись загрузки Windows для двойной загрузки, сначала войдите как root:

```bash
sudo su
```

Установите OS-prober:

```bash
pacman -S os-prober
```

Отредактируйте конфигурацию grub:

```bash
nano /etc/default/grub
```

Прокрутите вниз и раскомментируйте следующее

```bash title="grub" hl_lines="3"
#GRUB_SAVEDEFAULT=true
#GRUB_DISABLE_SUBMENU=y
GRUB_DISABLE_OS_PROBER=false
```

Нажмите ++ctrl+x++, ++y++ и ++enter++.

Теперь вам нужно проверить запись Windows Boot manager:

```bash
os-prober
```

Сгенерируйте файл конфигурации grub:

```bash
grub-mkconfig -o /boot/grub/grub.cfg
```

Должно быть всё хорошо, если отладочные сообщения говорят об этом.

Теперь точно ФИНАЛЬНАЯ перезагрузка:

```bash
reboot
```
